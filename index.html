Web Services

Syed Gillani, Fall 2016

Course Objectives

In general, web services are loosely coupled software components that are published, located and invoked across the Web. The main components of web services include: XML-RPC, SOAP protocol, WSDL and UDDI. These components are evolved from the basics of sharing information over the Web. Web services have become a mainstream development in the industry, where large enterprises (such as Google, Facebook, LinkedIn, AirBnB, etc.) are increasingly relying on them for large scale software development and sharing of services within an organisation. As a result, a multitude of applications are build together by piecing together web services published by third parties.

The main aim of this course is to introduce the underlying architecture of web services, i.e. data models, data formats, query languages and protocols, as well as a hands-on experience with building interesting web service-based applications. We will begin with the introduction of the data formats to share information on the Web and their respective query languages. We then study the protocols to discover the related web services and how to use them within various applications.
